Información JWT

JSON Web Token (JWT)

JWT es un estándar abierto (RFC 7519) que define una forma compacta y autosuficiente para transmitir información de forma segura entre partes como un objeto JSON.

¿Cómo funciona JWT?

  1. El usuario envía sus credenciales (usuario/contraseña) al servidor.
  2. El servidor verifica las credenciales y genera un token JWT firmado con una clave secreta.
  3. El token se devuelve al cliente y se almacena (en este caso, en una cookie HTTP-only).
  4. Para solicitudes posteriores, el token se envía automáticamente en cada petición.
  5. El servidor verifica la firma del token y, si es válido, procesa la solicitud.

Ventajas de JWT

  • Stateless: El servidor no necesita almacenar información sobre el token.
  • Transmisión segura: El token está firmado, lo que garantiza que no ha sido alterado.
  • Información embebida: El token contiene toda la información necesaria (como roles y permisos).
  • Escalabilidad: Ideal para arquitecturas distribuidas y microservicios.
